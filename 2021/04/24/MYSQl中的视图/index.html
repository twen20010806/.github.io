MySQL中的视图

发表于2021-04-24|更新于2021-04-24|MYSQL
|字数总计:848|阅读时长:2分钟|阅读量:

视图:VIEW

  • 含义: 虚拟表,和普通表一样使用
    • 保存的查询语句,可以直接调用,需要查询的时候直接调用视图
      • 类似于java的方法
    • mysql 5.1版本出现的新特性,是通过表动态生成的数据
  • 特点
    • 临时性:视图只有在被调用时虚拟表才会产生,没有保存数据,只保存了SQL逻辑,不保存查询结果
  • 应用场景
    • 多个地方用到同样的查询结果
    • 该查询结果使用的SQL语句较复杂
  • 好处
    • 重用了sql语句
    • 简化复杂的SQL操作,不必知道它的查询细节
    • 保护数据、提高安全性,只提供了一个接口,没提供原表

      视图的创建与使用

  • 案例
    1. 查询姓名中中包含a字符的员工名、部门名和工种信息
      • 创建
          CREATE VIEW 
      myv1
  AS
      SELECT 
      last_name,department_id,j.job_title
  FROM
      employees e
  JOIN    
      jobs j
  ON
      e.job_id = j.job_id;
      • 使用
          SELECT
      *    
  FROM
      myv1
  WHERE
      last_name LIKE "%a%";
    2. 查询各部门的平均工资级别
      • 创建视图查看每个部门的平均工资
          CREATE VIEW
      deparment_avg_salary
  AS
  SELECT    
      department_id,AVG(salary) ag
  FROM
      employees
  GROUP BY
      department_id
      • 使用
          SELECT
      department_id,grade_level
  FROM
      department_avg_salary d
  JOIN
      job_grades j
  ON
      d.ag BETWEEN j.lowest_sal AND j.highest_sal;
    3. 查询平均工资最低的部门信息
      • 创建视图:同2
      • 使用视图:同4
    4. 查询平均工资最低的部门名和工资
      • 创建视图:同2、3
      • 使用视图
          SELECT
      *
  FROM
      department_avg_salary
  WHERE
      ag = (
  SELECT
      MIN(ag)
  FROM
      department_avg_salary);

        视图的修改

  • 方式一
    • 语法
        CREATE OR REPLACE VIEW 
      视图名
  AS    
      查询语句；
    • 作用
      • 创建或者改变视图
        • 视图不存在创建
        • 视图存在改变
  • 方式二
    • 语法
        ALTER VIEW
      视图名
  AS
      查询语句
    • 作用
      • 改变视图,不能创建视图

视图的删除

  • 语法

      DROP VIEW
      视图名,视图名;

    视图结构的查看

  • 语法

    1. 展示视图结构,和展示表结构类似
       DESC
     视图名;
    2. 展示详细信息(创建语句,视图名,字符集等等)
       SHOW CREATE VIEW
     视图名;

      视图的更新

  • 视图不允许更新的特点

    1. 具有以下关键字的SQL语句:分组函数、distinct、group by、having、union、union all

      • 不是原表原数据的数组不允许更改

    2. 常量视图

    3. SELECT 中包含子查询

    4. SELECT 中包含join,92,99语法都不行

    5. FROM一个不能更新的视图

    6. WHERE子句的子查询引用了FROM子句中的表

  • 总结

    • 视图更新时必须只能参照原表的数据,不能更新由原表数据进行任何操作形成的视图。
    • 视图的更新会对原表的数据进行更新,这就是为什么视图更新只能参照原表的数据。
    • 对于创建视图时引用了不能跟新的视图时也不能更新,级联更新
    • 视图的更新只能对单个表,单行数据更新
    • 视图中不允许更新可能是不允许CRUD中的某一项也可能都不支持

视图和表的对比

* 创建语法
    * CREATE VIEW
    * CREATE TABLE
* 是否占用物理空间
    * 视图不占用数据存储空间,占用逻辑语句存储的空间,只保留了SQL逻辑
    * 表占用数据空间
* 使用
    * VIEW可以使用增删改查,但一般不用,且使用起来有限制
    * TABLE使用见DML
